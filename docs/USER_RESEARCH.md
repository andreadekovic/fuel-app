# FUEL User Research

## Research Goal

Validate whether project-based teams have a painful, frequent, and valuable problem around receiving one client payment and splitting it across collaborators.

The goal is not to ask whether people like FUEL. The goal is to understand how they currently coordinate money, where it breaks, and whether the FUEL workflow would save enough time, stress, or fees to justify adoption.

## Target Interviewees

Prioritize:

- Creative agency owners.
- Freelance collectives.
- Project leads who hire multiple contractors.
- Video, design, dev, music, or production teams.
- Cross-border contractor teams.
- Web3 agencies that work with non-crypto collaborators.

Avoid early:

- Traditional payroll buyers.
- Large finance teams with heavy procurement requirements.
- Users who only pay one contractor at a time.

## Screening Criteria

Good interview candidates should answer yes to at least two:

- They receive client payments for collaborative work.
- They pay multiple people from one project budget.
- They work across countries or currencies.
- They use spreadsheets or manual calculations for splits.
- They have had payout delays, mistakes, disputes, or fee frustration.

## Interview Script

### Warmup

1. What kind of project work do you do?
2. Who do you usually collaborate with?
3. Who receives the client payment?
4. Who is responsible for paying everyone else?

### Current Workflow

1. Walk me through the last time you had to split a project payment.
2. What tools did you use?
3. How did you calculate each person's amount?
4. How did you actually send the money?
5. How long did the whole process take?
6. What fees did you pay?
7. What records did you keep?

### Pain

1. What is the most annoying part of the workflow?
2. What has gone wrong before?
3. Have collaborators ever waited longer than expected?
4. Have you ever made a mistake in a split?
5. Do you avoid certain cross-border collaborators because payment is annoying?
6. How embarrassing or stressful is the payment coordination part?

### Buying Signal

1. If a tool automated the split and payout tracking, when would you use it?
2. What would make you trust it?
3. What would make you not trust it?
4. Would you pay a transaction fee for this? If yes, what range feels acceptable?
5. Would you rather pay per transaction or monthly?
6. What would need to be true before you connected real money?

### Product Reaction

Show the prototype after understanding their workflow.

Ask:

1. Which part feels most useful?
2. Which part is confusing?
3. What is missing before you could use this?
4. Would you invite collaborators into this?
5. What would the first project be?

## Validation Signals

Strong signals:

- User already pays multiple collaborators from project revenue.
- User can describe a recent painful payout story.
- User uses spreadsheets/manual calculations.
- User has paid high fees or waited days.
- User asks if they can try it.
- User offers to introduce another team.

Weak signals:

- User says it is interesting but has no recent example.
- User only pays one person at a time.
- User delegates all payment work to payroll/accounting.
- User only wants a cheaper Wise alternative.
